## 0. Setup & finding

- API server PID 59191 started **2026-05-19 02:03:19**.
- Commit that adds `profile_changed` emission + migration 065: `3c62e9df` **at 02:29:41**.
- Live SQLite (`agent-swarm-db.sqlite`) is **at migration 064**, not 065.
- Direct INSERT of `eventType='profile_changed'` against the live DB rejects:
  ```
  CHECK constraint failed: eventType IN ('auto_merge','manual_merge','identity_added',
  'identity_removed','email_added','email_removed','token_minted','token_revoked',
  'budget_changed','status_changed')
  ```
- The PATCH route does call `recordIdentityEvent("profile_changed", …)`, but the running server is from before that code shipped — so the route silently no-ops for the profile-fields branch and the constraint never fires in practice. After `bun run pm2-restart` (or any normal redeploy), migration 065 will auto-apply and the event will land.
- **Action**: deploy notes should mention "restart the API server so migration 065 applies" — not technically a code bug, but worth flagging since the running server I tested against was the obvious thing in front of me, and a freshly-merged dev environment will produce the same gap until restart.

### BUG-1 reproduction trail (exact request)

```http
POST /api/users/unmapped/slack/%40qatest/resolve HTTP/1.1
Authorization: Bearer 123123
Content-Type: application/json

{"name":"QA AtTest","email":"atqa@example.com"}
```

Server response: `200 OK`.

DB state after:
```
user_external_ids:  {userId: ca3aa7d0…, kind: "slack", externalId: "%40qatest"}
kv_entries:         integration:unmapped:slack — @qatest:meta + @qatest:count still present
GET /api/users/unmapped?kind=slack → still lists {externalId: "@qatest", count: 4}
```

### BUG-1 fix

Branch: `fix/url-decode-identity-params` (off `feat-human-first`, **not pushed**).

Commit: `35fdd982 fix(http): decode externalId path param in user resolve + unlink`.

- `decodeURIComponent` applied to the externalId path param in:
  - `src/http/users.ts` resolveUnmapped handler (POST `/api/users/unmapped/{kind}/{externalId}/resolve`)
  - `src/http/users.ts` deleteIdentityRoute handler (DELETE `/api/users/{id}/identities/{kind}/{externalId}`)
- New regression tests in `src/tests/http-users.test.ts`:
  1. `POST /api/users/unmapped/:kind/:externalId/resolve > URL-encoded externalId is decoded — kv rows clear, identity stored decoded` — exercises `@alexdev` via `encodeURIComponent` end-to-end through `Bun.serve` + fetch.
  2. `identity link/unlink > DELETE with URL-encoded externalId removes the literal identity` — re-fetches the user after delete to prove the literal-keyed row is actually gone (the prior pre-fix code would have returned `identities: []` while the row stayed in the DB).
- All 19 tests in `src/tests/http-users.test.ts` pass on the fix branch; `bun run lint` clean; `bun run tsc:check` clean.

Decision left to Taras: roll the fix commit into PR #500 (cherry-pick into `feat-human-first`) or merge separately after #500.
