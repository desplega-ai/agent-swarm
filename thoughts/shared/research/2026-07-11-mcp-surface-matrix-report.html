Analysis & recommendations

1 · The seed pack closed the Claude gap completely

Unseeded code-mode cost $3.13 / 9.2 min; with the 6 seed scripts + fixed prompt it dropped to $1.85 / 3.3 min — cost parity with the full surface ($1.83) and faster wall time (3.3 vs 3.9 min), with 3/3 runs completing and delegating correctly. Claude agents adopted the seeds heavily (16 seed invocations/run) and SDK-probing vanished. Context is where code-mode now wins outright: lead sessions peaked at ~25K vs ~39K tokens (−37%), workers ~22K vs ~29K, because coordination logic lives in scripts instead of conversation turns.

The remaining 5 "errors"/run are one trivial bug: the prompt tells agents to call script-run, but Claude Code registers it as mcp__agent-swarm__script-run — agents try the bare name once, fail, adapt. One prompt line ("tool ids may carry an mcp__agent-swarm__ prefix") removes them.

2 · The schema-bloat hypothesis confirmed on opencode

Without tool-search, the 118-tool surface genuinely enters context: opencode/full sessions peaked at ~75–83K tokens — roughly the predicted ~80K of schema. Scripts-only cut the (one completed) lead session to ~38K (−55%). The context win is real exactly where it was predicted to be. (pi doesn't report context usage at all — an adapter gap worth fixing regardless of this experiment.)

3 · But small models break as agents in code-mode

deepseek-v4-flash handled the full surface fine: pi/full went 3/3 completed and correctly delegated, in 1.9 min at $0.04/run. In scripts-only mode the same model kept "completing" tasks while quietly dropping the actual job: across pi+opencode scripts-only runs, only 1 of 3 completed runs delegated to both workers — leads did the work solo, skipped the marketer, or lost parent-task lineage (children created without parentTaskId). pi's lead also ballooned to ~150 tool calls/run (including 103 bash calls) and ignored the seed catalog entirely (0 seed invocations). Writing correct coordination code on the fly is a capability threshold that deepseek-flash-class models don't clear, even though they operate named tools perfectly well.

Recommendations

  • Claude agents: scripts-only + seeds is now a legitimate default — cost parity, faster, and ~37% more context headroom for long sessions. At minimum, ship the seeds: they help the hybrid mode too.
  • pi/opencode with small models: keep the full surface for now. The 80K schema tax is real, but code-mode currently trades it for broken delegation fidelity — a worse failure mode. Revisit with a stronger model or with coordination seeds hardened into the task protocol itself.
  • Ship regardless: the 6 seed scripts as global defaults, the code-mode prompt (with exact prefixed tool ids), typed returns in swarm-sdk.d.ts, and pi context-usage reporting.
  • Grade delegation, not just completion: "parent completed" was a misleading success signal in 2 runs — the delegation-fidelity check (both workers received non-review tasks) belongs in the evals harness.

Caveats: n=3 per cell (n=2 for pi/scripts-only after one docker-compose flake; opencode had 1 boot timeout + 2 "superseded" crash-recovery outcomes), single simple task, one small model for pi/opencode, Claude costs are subscription estimates while deepseek is metered — compare within provider only.
