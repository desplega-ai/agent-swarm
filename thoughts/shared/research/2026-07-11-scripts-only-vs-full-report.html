Analysis

Verdict: for a simple delegation task on the Claude harness, the full tool surface wins decisively. Scripts-only cost +71% ($3.13 vs $1.83), took 2.4x longer (9.2 vs 3.9 min), made 2x the tool calls (49 vs 25), and hit 11x more tool errors (3.7 vs 0.3 per run). Every scripts-only run completed correctly — the architecture works — but it pays a steep boilerplate tax.

The expected context win did not materialize on Claude. End-of-session context was a wash (workers ~27K scripts-only vs ~29K full; leads ~37K vs ~39K). Claude's ToolSearch already defers the 118 tool schemas (~80K tokens) out of context, so trimming the surface to 8 tools saved little in practice. Worse, scripts-only agents write TypeScript as output: 2.5x the output tokens (21.9K vs 8.9K) — and output tokens are the most expensive kind. That, plus SDK-probing (5.7 scratch scripts per run discovering the (args, ctx) signature and response envelopes) and hand-rolled sleep/poll loops, fully explains the cost gap.

Where the 35x schema reduction still matters:

  • Harnesses without tool-search (pi, codex, opencode) eat the full ~80K schema payload in every session. Scripts-only is likely a genuine context/cost win there — untested in this matrix, worth a follow-up.
  • Bulk/fan-out work. This scenario had zero bulk operations; every SDK call was 1:1 with a tool call, which is exactly where code-mode is weakest. The existing script rubric (scripts for 10+ item pipelines) remains the right call.

The gap is boilerplate-shaped, not architecture-shaped. Probing, storeProgress/delegation boilerplate, and wait loops account for roughly half of scripts-only tool calls and most of the extra output tokens. All are addressable: document the entry signature in the prompt, add typed returns to swarm-sdk.d.ts, and ship default seed scripts (delegate, wait-for-task, get-child-outputs, complete-task, swarm-overview). A re-run with that pack would show whether the gap closes.

Recommendation: don't strip the named tools for Claude agents — keep the hybrid (full surface + script rubric for bulk work). Revisit scripts-only as the default surface for non-tool-search harnesses once the seed-script pack + typed SDK land, and consider a middle surface (8 script tools + the ~10 core task-lifecycle tools) as the best of both.

Caveats: n=3 per mode, one simple task, Claude harness only, cost figures are harness-reported estimates; one scripts-only attempt lost to an infra boot timeout (excluded, $0 spent). Scripts-only run 2 included a ~10 min wait-loop stall (lead polling a finished child on a slow cadence) — its wall time is a fair reflection of the pattern, not an outlier artifact.
