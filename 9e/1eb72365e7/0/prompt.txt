### Design decisions

1. **`upsertPromptTemplate` insert path**: New templates get `isDefault=0` (false). The only way to set `isDefault=true` is via `resetPromptTemplateToDefault`. This avoids accidental creation of "default" templates through the normal upsert flow.

2. **History on insert**: The initial creation also gets a history entry with `changeReason: "Initial creation"` (when no explicit reason is provided). This ensures every version is tracked.

3. **`resolvePromptTemplate` return type**: Uses a discriminated union `{ template: PromptTemplate } | { skip: true } | null` to clearly distinguish the three outcomes (found template, skip event, no match).

4. **Wildcard generation**: For `a.b.c.d`, generates `["a.b.c.*", "a.b.*", "a.*"]` -- narrower wildcards are tried first, and each wildcard goes through the full agent -> repo -> global scope chain before moving to the next broader wildcard.
