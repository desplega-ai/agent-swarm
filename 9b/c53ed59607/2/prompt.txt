# Plan: Optimize Dockerfile.worker & docker-entrypoint.sh

## Context

The Docker worker image rebuilds slowly because:
1. `@latest` packages (`wts`, `qa-use`) bust the npm cache on every build
2. Claude marketplace plugin installs happen at **runtime** (entrypoint) instead of build time
3. Static directory creation and wts config happen at runtime unnecessarily
4. apt-get is split across 3 separate RUN commands (3 layers instead of 1)

Reference: [exeuntu Dockerfile](https://github.com/boldsoftware/exeuntu/blob/main/Dockerfile) — single consolidated apt layer, version pinning, clean layer ordering.

## Changes

### Phase 1: Pin package versions in Dockerfile.worker

**File**: `Dockerfile.worker` (lines 106-112)

Replace `@latest` with pinned versions:

### Phase 2: Consolidate apt-get into a single RUN

**File**: `Dockerfile.worker` (lines 27-57)

Merge the 3 separate `apt-get update && apt-get install` blocks (base packages, Node.js, GitHub CLI) into a single RUN command. This:
- Reduces layers from 3 to 1
- Avoids redundant `apt-get update` calls
- Follows exeuntu pattern

The consolidated RUN should:
1. Add NodeSource and GitHub CLI apt repos first (just the key + sources.list)
2. Run a single `apt-get update`
3. Install everything in one `apt-get install -y`
4. Single `rm -rf /var/lib/apt/lists/*`

```dockerfile
RUN curl -fsSL https://deb.nodesource.com/setup_22.x | bash - \
    && curl -fsSL https://cli.github.com/packages/githubcli-archive-keyring.gpg \
       | dd of=/usr/share/keyrings/githubcli-archive-keyring.gpg \
    && chmod go+r /usr/share/keyrings/githubcli-archive-keyring.gpg \
    && echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/githubcli-archive-keyring.gpg] https://cli.github.com/packages stable main" \
       | tee /etc/apt/sources.list.d/github-cli.list > /dev/null \
    && apt-get update && apt-get install -y \
    curl wget ca-certificates gnupg lsb-release \
    git git-lfs \
    vim nano \
    build-essential make cmake gcc g++ \
    python3 python3-pip python3-venv \
    jq tree htop unzip zip tar gzip sqlite3 \
    openssh-client \
    sudo \
    nodejs \
    gh \
    && rm -rf /var/lib/apt/lists/*
```

**Verification**: `docker build -f Dockerfile.worker -t agent-swarm-worker:test .`

### Phase 4: Optimize layer ordering in Dockerfile

Reorder for maximum cache hits (least-changing first):

1. Base OS + all apt packages (consolidated - rarely changes)
2. Bun install (root + worker) — rarely changes
3. User creation + sudo
4. Claude CLI install
5. Marketplace plugins (new, cached after first build)
6. Static config (wts, directories)
7. Claude settings.json + hooks config
8. Global npm packages (pinned versions — cached until manual bump)
9. **Binary from builder** (changes on every code change)
10. Plugin commands/agents/skills COPY (changes on code change)
11. Entrypoint COPY (changes on code change)

Key insight: the binary COPY and plugin file COPYs should come **after** all package installs since they change most frequently.

**Verification**: Change a source file, rebuild — only steps 9-11 should re-run.
